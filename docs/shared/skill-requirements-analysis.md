!!! info "来源"

## 概述

[requirements-analysis](https://clawhub.ai) 是一个**需求分析专家 Skill**，能够通过对话式引导，将一句话的模糊需求转化为完整、结构化的需求规格说明。

- **安装方式**: ClawHub 一键安装
- **类型**: 纯 Markdown Skill（无脚本/代码）
- **协议**: MIT-0
- **安全审查**: ClawHub CLEAN ✅

## 安装

```bash
npx clawhub@latest install requirements-analysis
```

安装后 Skill 会出现在 workspace 的 skills 目录中，Agent 会自动识别并在需求分析场景下激活。

## 核心功能

### 🗣️ 对话式需求提取

不需要你写一份完整的 PRD，只要说出大概想法，Skill 会通过**系统化的问题**逐步引导你补全：

- 目标用户是谁？
- 核心问题是什么？
- 成功标准怎么定义？
- 有哪些约束条件？

### 📐 分层需求分解

按照行业标准的层级结构拆解需求：

```
EPIC（史诗故事）
  └── 需求（Requirement）
       └── 用户故事（User Story）
            └── 验收标准（Acceptance Criteria）
```

用户故事遵循 **INVEST 标准**：

| 原则 | 含义 |
|------|------|
| **I**ndependent | 独立的，不依赖其他故事 |
| **N**egotiable | 可协商的，不是合同条款 |
| **V**aluable | 有价值的，对用户有意义 |
| **E**stimable | 可估算的，能评估工作量 |
| **S**mall | 小的，一个迭代内能完成 |
| **T**estable | 可测试的，有明确验收标准 |

### 🎯 优先级排序

支持多种业界优先级排序方法：

- **MoSCoW**: Must / Should / Could / Won't
- **RICE**: Reach × Impact × Confidence ÷ Effort
- **Kano**: 基本型 / 期望型 / 兴奋型

### ✅ 验收标准

使用 **Given-When-Then** 格式生成清晰的验收标准：

```gherkin
Given 用户已登录系统
When 用户点击「创建项目」按钮
Then 系统显示项目创建表单
And 表单包含项目名称、描述、截止日期字段
```

### 📄 文档生成

自动生成包含以下内容的完整需求文档：

- 项目概述与目标
- 核心概念模型
- EPIC 分解
- 用户故事列表
- 技术约束与非功能需求
- 验收标准汇总

## 实际体验

!!! success "NEKO 小队实践反馈"
    主人用这个 Skill 做了 EpicPoet 项目的需求分析，通过 **3 轮对话**就产出了完整的需求文档，覆盖：

    - 核心概念模型
    - EPIC 分解
    - 用户故事（含验收标准）
    - 技术约束

    整个过程对话式引导非常自然，不需要提前准备任何模板。

## 注意事项

- Skill 本身是纯 Markdown，**不会执行任何代码**，安全性高
- 需求分析的质量取决于你提供的上下文信息——越具体越好
- 生成的文档是起点，不是终点，后续还需要根据实际情况调整
